# KDDTAAC 比赛入门指南

这份文档是给当前项目的入门说明，目标不是一次性把比赛讲到很深，而是帮你用最小阻力开始动手。

## 1. 这是什么比赛

根据腾讯广告算法大赛官方公开介绍，本届题目方向是“全模态生成式推荐”。对新手来说，可以先把它理解成一个更容易上手的问题：

- 输入：用户特征、物料特征、用户历史行为序列、时间信息
- 输出：预测用户是否会发生某种目标行为

不要被“生成式”“全模态”这些词吓到。比赛名字可以很前沿，但入门时完全可以先当成一个监督学习任务来做。

## 3. 你现在应该先学什么

如果你是小白，不建议一上来就追复杂模型。最适合的顺序是：

1. 学会读数据
2. 学会看标签和特征
3. 先跑一个最简单的 baseline
4. 再逐步增加特征和模型复杂度

对应的知识点如下：

- `pandas` / `pyarrow`：读取 parquet 数据
- `python` 基础：字典、列表、循环、函数、文件路径
- 机器学习基础：训练集、验证集、特征、标签、过拟合
- 表格建模基础：LightGBM、XGBoost、CatBoost 三选一先学一个

## 4. 数据可以先怎么理解

公开示例数据集中可以看到这些核心字段：

- `user_id`
- `item_id`
- `timestamp`
- `label`
- `user_feature`
- `item_feature`
- `seq_feature`

其中最重要的是三类信息：

- `user_feature`：用户本身的特征
- `item_feature`：候选内容或广告物料的特征
- `seq_feature`：用户过去的行为序列

你可以把它理解成：

“给定一个用户、一个候选物料，以及这个用户过去做过什么，预测这次会不会点/看/转化。”

## 5. 新手最稳的比赛路线

### 第一步：先做 EDA

EDA 就是探索性数据分析。你需要先回答这些问题：

- 每一列是什么类型
- `label` 是 0/1 还是多分类
- 有没有缺失值
- 序列字段平均长度是多少
- 不同 `feature_id` 出现频率如何

这一阶段不要急着训练模型。先把数据看懂，比盲目调参更重要。

### 第二步：做最小 baseline

一开始的 baseline 可以非常简单：

- 统计用户特征数量
- 统计物料特征数量
- 统计历史行为长度
- 提取时间特征，如小时、星期几
- 把这些特征送进一个树模型

目标不是一开始拿高分，而是先打通下面这条链路：

数据读取 -> 特征处理 -> 划分验证集 -> 训练模型 -> 输出结果

### 第三步：逐步增加特征

当最小 baseline 跑通以后，再慢慢增加：

- 最近 N 次行为统计
- 不同行为类型计数
- 行为时间间隔
- 高频特征交叉
- 用户和物料的历史统计特征

### 第四步：再考虑更高级的方法

等你把传统 baseline 跑通以后，再考虑：

- 序列模型
- embedding
- 多模态融合
- 更贴近题目包装的生成式建模思路

## 6. 第一周怎么学最合适

### Day 1

- 激活环境
- 安装基础依赖

- 统计字段分布
- 看缺失值

- 构造几个简单特征

- 跑一个最基础模型

### Day 4 到 Day 7

- 补更多统计特征
- 对比不同验证方案
- 整理实验记录
- 形成自己的 baseline 版本

## 7. 你每天真正需要做的事

很多新手学比赛时容易陷入两个误区：

- 一直看教程，不动手
- 一上来就看复杂方案，结果基础没打牢

更有效的方法是每天坚持做三件小事：

1. 读一点点理论
2. 改一点点代码
3. 记录今天学到了什么、跑了什么结果

只要连续一周，你会比现在清楚很多。
